# ETL — Sakila OLTP to Movie Rental DW

This folder contains the practical ETL layer for the Data Warehouse project.

It follows the instructor's style:

```text
Extract  → read from OLTP tables
Transform → clean, join, calculate keys/measures
Load     → insert into DW dimensions and facts
```

## Main file

```text
sakila_etl.py
```

## Before running

1. Import the Sakila OLTP dump as database `sakila`.
2. Run the GitHub schema file:

```text
sql/create_dw_schema.sql
```

3. Install requirements:

```bash
pip install -r ETL/requirements.txt
```

4. Create `.env`:

```text
OLTP_URI=mysql+pymysql://root:YOUR_PASSWORD@localhost:3306/sakila
DW_URI=mysql+pymysql://root:YOUR_PASSWORD@localhost:3306/movie_rental_dw
```

## Run

```bash
python ETL/sakila_etl.py
```

## Loading order

1. `dim_location`
2. `dim_language`
3. `dim_category`
4. `dim_actor`
5. `dim_film`
6. `bridge_film_actor`
7. `dim_store`
8. `dim_staff`
9. `dim_customer`
10. `fact_rental`
11. `fact_payment`

`dim_date` is already generated by the GitHub SQL schema procedure.
